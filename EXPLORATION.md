## 0. 位置づけ（前提）

- これは **自主制作の実験作品**。展示・プロダクトリリースの予定はない。
- 成果物は **動画作品（シリーズ）**。リアルタイムで観衆に届ける用ではない。
- DJ/VJ 的な公開パフォーマンスではない。アンビエント音 ＋ G2 の私的さを活かした **内的・親密な体験**。
- 多くの人に長く使わせる継続利用の設計は目的ではない。狙いは **コンセプトの探求と、撮影での記録**。

### 最初のスコープ（ミニマム）

> まずは **自然 archetype（公園）単体** で完成度を上げる。GPS 自動判定（§4）と他 archetype（都市・水辺）は **ロードマップに置いて後回し**。
> 自然1本に集中することで音／視覚／インタラクションの**手触りを詰める**。決まった「触れる体験の語彙」が、後の archetype 展開の土台になる。

## 1. コンセプト（核）

**「私と世界の私的通信」**

触れることで、世界の隠れた“声”と、自分だけの回線（G2）が開く。単に音が鳴るのではなく、**世界と双方向に交信している**感覚。

> **「で、これは何？」への答え**:
> 「世界の異なる場所で、自分だけが感じ取った私的な交信の記録」。
> 身のまわりに生きている世界に、自分だけが触れて気づく実験。

---

## 2. 成果物のかたち：動画作品のシリーズ

- **作家本人が撮影者かつ体験者**。Playtron＋G2 を持って実際の場所に赴き、触れ、その様子を映像にする。
- **1地点 15〜20秒程度 × 3地点** を編集して 1 作品（〜1分）。複数本のシリーズを想定。
- 各クリップは独立した「世界のチャンネル」からの受信記録。
- 集合体として「Conduct field recordings」のような連作になる。

> §5 の循環設計と §6 のインタラクション語彙（残響モデル含む）が、短いクリップ内でも世界の呼吸と触れの掛け算で**毎回違う表情**を作る。同じ場所を別の日に訪れても同じ交信にならない。これが**作品としての価値**になる。

## 3. 何と交信するか：「場所」が交信相手

### 軸の組み合わせ

- **自然物それぞれの “声”（核）**: 物質ごとに固有の応答を持つ
- **時間・自然のリズム（採用）**: 世界の側が独自の時間で動いている
- これらを **「場所＝archetype」** に統合する。場所が、その時点での世界のチャンネルを決める

### 3 archetype（最初は自然のみ）

| | archetype | スコープ | 触れる対象（例） | 通電のヒント |
|---|---|---|---|---|
| **A** | **自然 (forest/park)** | **★最初に作る** | 木の皮／土／苔／落ち葉の地面 | 湿った土・苔ほど強い／乾燥は弱い |
| **B** | **都市 (urban)** | 後回し | 金属手すり／マンホール／自販機の枠／街灯ポール／金属フェンス | コンクリ単体は不安定 → 金属を核に |
| **C** | **水辺 (waterside)** | 後回し | 水中にクリップを垂らす | ほぼ完璧。海水は淡水より強い |

> 撮影地・触れる対象の最終確定は **事前ロケハン＋通電テスト** で決める（§9）。

### 「都市」archetype のコンセプト的フレーム（将来用に保管）

都市 ＝ **見えない金属神経網に触れる**、と捉える。コンクリの皮膚の下にある鉄筋・配管・電線・換気・地下水路 — これらは**都市の隠れた身体**。普段は意識されない。Conduct で触れることで「**都市が生きている＝鉄でできた生体**」として通信される、という詩。3地点の中で**都市が一番非自明な発見**になる。

### 都市のコンクリ通電について（参考・将来用）

乾燥した純コンクリは半絶縁体だが、現実のコンクリ（粉塵・湿気・劣化）はそこそこ通電する。Playtron は閾値が低めなので**動くことはある**。ただし「**金属より明らかに弱く・不安定**」。よって:

- コンクリ単体には頼らない
- やりたい絵があるなら、雨後 or 軽く水を含ませると実用域に
- 安定なのは「**コンクリの中の／に付随する金属**」（手すりの根元・ガードレールのボルト・側溝の蓋）

## 4. GPS で「場所」を読み取る（hybrid 方式・ロードマップ）

> **このセクションはロードマップ用**。最初の自然 archetype 単体スコープではまだ実装しない。archetype を増やすタイミングで導入する。

### 仕組み

| | 役割 | 手段 |
|---|---|---|
| 1. 位置取得 | 緯度経度を取得 | `navigator.geolocation`（ユーザー許可・HTTPS or localhost 必須） |
| 2. 周辺地物の照会 | 近くに何があるか | **OpenStreetMap** の **Overpass API** または **Nominatim**（無料・APIキー不要） |
| 3. 分類ロジック | archetype を決める | ルールベース |

### 分類ルール例（100〜200m 圏内）

| archetype | OSM タグ条件（優先順） |
|---|---|
| **水辺** | `natural=water` / `waterway` / `natural=coastline` が近接（〜50m） |
| **自然** | `leisure=park` / `landuse=forest` / `natural=wood,grassland` が dominant |
| **都市** | 上 2 つに該当せず、`building` `highway` 密度が高い（or 既定） |

### ハイブリッド方式

- 撮影前に**装置が自動で archetype を推測**して提示
- 表示例: `[scanning... lat 35.31 / lng 139.59 / detected: 水辺]`
- 撮影者は **承認 or 手動上書き**
- 事前判定の結果をセッションにキャッシュ（毎回 GPS を叩かない）

> **完全自動 1択にはしない**。コントラストをくっきり出したい時など、作家の意図が優先される経路を常に残す。

### コンセプト的に効くポイント

- 装置が「**今ここを読み取った**」という所作が、「世界との私的通信」を**もう一段リアル**にする
- 動画で `scanning… → detected: 水辺` の瞬間を映せば**ドキュメンタリー的厚み**が出る → **作品の見せ場の一つ**になる
- 各クリップに緯度経度＋archetype が**メタデータ**として刻まれ、作品が「**世界の特定の座標からの受信記録**」として固有のアイデンティティを持つ

### 注意点

- **HTTPS 必須**（localhost は OK）
- ユーザー許可ダイアログが必ず出る → 撮影前セットアップに含める
- Overpass API はレート制限あり・タイムアウト推奨 5秒
- 屋内は精度悪化（〜20m ずれ）

## 5. 中心テーマ：循環する世界

世界の側が**長期的に変化しつつ循環している**。触れることは「その**今**に同調する」こと。二度と同じ交信にならない。

### 採用する設計（3つの組み合わせ）

#### ① 呼吸／潮（tide）モデル
長周期（5〜15分）の slow swell＋recede。世界が独自に呼吸している。**低潮 ≠ 死**、静謐な「負の空間」として価値化する。

#### ② 周期の重ね合わせ（非可換）
短／中／長の周期を**互いに割り切れない長さ**で重ねる。位相がずれ続けて完全には反復しない。

#### ③ 触れることが波を押し上げる
世界の波 ＋ 触れたぶんも波に足される。**沈黙の時に触れる ＝ 何かを呼び起こす**。**高潮の時に触れる ＝ 開かれた世界に深く入る**。

> 「盛り下がり」は**呼吸の負の空間**。受け身の循環ではなく、ユーザーの介入と循環の合わせ技で起伏が生まれる、と発想転換。

### 待機（untouched）の再解釈

無タッチ時のうっすらビープは「**世界の息遣い**」と再解釈する。世界が独自に生きている／呼吸している、その小さな信号。

## 6. インタラクションの語彙（"会話" の作り方）

「ただ触れる → 音が鳴る」だけだと薄い。だが DJ/VJ にはしたくない。
→ **「会話」** として設計する: 振る舞いの違い（短い／長い／連続／重なる／間）で世界の応答が変わる。さらに**過去の振る舞いが残響として鳴り続け、新しい振る舞いと干渉する**ことで、対話に**文脈**が生まれる。

### 6.1 振る舞いと応答の対応（5+1）

| | 振る舞い | 検出 | 世界の応答 |
|---|---|---|---|
| **挨拶** | 一瞬触れて離す | <300ms の short tap | 短い ping。小さな glyph が一瞬光る |
| **問いかけ** | 連続で叩く | 3〜5 回 / 500ms 間隔以内 | melody が立ち上がる。glyph が連なる |
| **耳をすます** | 長く触れ続ける | >3s hold（既存 develop） | development 育成。glyph が滲み広がる |
| **会話の取り戻し** | 数秒空けてまた触れる | last release から 5〜30s 後の touch | 続きを始める／前のテーマが残り香で戻る |
| **重なる声** | 複数同時に触れる | 同時 hold ≥2 | 和音的に厚く。glyph 同士が線で繋がり**星座** |
| **間** | 触れずに置く | no touch ≥10s | 世界の呼吸が広がる。長い波 |

### 6.2 反芻（echo / decay / interference）— 鍵となる設計

> 一回のタッチの演出が完結してから次が始まる、では味気ない。**過去の振る舞いが残響として鳴り続け、新しい振る舞いと干渉する**設計にする。これにより**会話の文脈が持続**する。Looper/Delay の比喩は **musical context が連続する** ことを言っており、performance ではなく **conversational continuity** を作るための道具。

#### 残響モデル

各振る舞いは、検知時に**残響オブジェクト**を残す:

| 振る舞い | 残響の内容 | 減衰 |
|---|---|---|
| 挨拶 | 短く小さな波紋（音と glyph） | 数秒で消える |
| 問いかけ | ループのフレーズ片を残し繰り返しながら減衰 | 10〜20秒で消える |
| 耳をすます | 触れていた音域・密度が場に残る | 触れていた時間に比例し、ゆっくり消える |
| 重なる声 | 形成された星座／和音が宙に残る | 中〜長期 |

これらは時間とともに **音量・密度・存在感が減衰**するが、ゼロにはならず**世界の文脈**を作る。

#### 干渉（interference）

新しい振る舞いは、残響している過去の振る舞いと**重なり相互作用**する:

- 過去の問いかけが残っているところに新しい挨拶 → 短い ping が問いかけの拍に同期して入る
- 長押し中に短く別の場所をタップ → 長押しのテクスチャの上に瞬きが乗る
- 残響が薄れた頃に新しい行動 → 新しい文脈の始まりとして響く

#### なぜ "演奏" にならないか

確かに**少し演奏的な要素**を持つが、本質は会話のまま:
- 精度を要する操作はない
- 残響は**システムが音楽的に処理**する（自分の拍が音楽の拍にはならない）
- 失敗概念がない
- 重要なのは**「対話の流れ」**であって**「正しい弾き方」**ではない

### 6.3 実装の方針

エンジンに「ジェスチャ分析レイヤー」を足す:

- `touchHistory: {note, t_on, t_off?, kind?}[]` を保持
- noteon/noteoff の時間パターンから kind（挨拶／問いかけ／耳をすます／重なる声）を判定
- kind ごとに**残響オブジェクト**を生成（楽器パートのパラメータ、ループフレーズ、glyph 群）
- 時間とともに減衰
- 既存の生成エンジン（§5 の循環）に残響を**重ねて出力**
- 視覚側も同じ kind 情報を受け取り、残響に合わせて glyph を残す

## 7. 音と視覚：archetype ごとの palette と表現

### 7.1 音の hint（叩き台）

| | スケール／モード | 音色・密度 | 印象 |
|---|---|---|---|
| **自然**（★最初） | ペンタ／ドリアン | 木質パッド・空気・鳥的高音点描 | 木洩れ日・森の奥 |
| **都市** | マイナー寄り／全音音階の混合 | 金属 FM・遠いシンセ・電子点描 | 鉄でできた生体・ネオン・余韻 |
| **水辺** | 5度音／低音多め | 液体パッド・低ドローン・波動 | 波・潜水・水底 |

> 完全に別ジャンルにはせず、**アンビエントの中で felt different なパレット**を切り替える。違和感のない切り替え、しかし耳で確かに分かる差。

### 7.2 ビジュアルの方向：「世界から届く私的な信号」

VJ ではなく**受信／通信の美学**。G2 の緑モノクロ＝レトロ受信機・ターミナル美学が、コンセプトに**ドンピシャ**で噛む（制約が逆に強み）。

#### 採用アプローチ

- **受信／ターミナル**: ASCII テキストが**信号のように到来**。物に触れると、その物固有の glyph 列が流れる。たまに単語/フラグメントが滲む。
- **星座／象形**: 物ごとに固有の glyph（自分専用の象形文字）。触れると glyph 同士が線で繋がり**星座**になる。
- → **両者の組み合わせ**で「**自分だけの言語で世界と交信**」感を作る。

### 7.3 G2 描画方式：テキスト＋画像のハイブリッド（採用）

> フレーム連発（〜150ms / フレーム）で**コマ撮りアニメ的なカクッ**を作る。私的な信号・装置感に合う。

#### G2 表現手段（おさらい）

| 手段 | 解像度・更新 | 用途 |
|---|---|---|
| **テキストコンテナ** (`textContainerUpgrade`) | 576×288、最大 2000 文字、〜100〜200ms 毎更新可（5〜10fps） | ASCII / グリフ / 文字信号 |
| **画像コンテナ** (`updateImageRawData`) | 各 200×100 max、4-bit greyscale（16階調） | ドット・抽象図形・波形 |

#### ハイブリッド構成

- **背景レイヤー（テキスト）**: 全画面に薄い ASCII の漂い（霧・点描・流れる文字信号）。世界の息遣い
- **前景レイヤー（テキスト）**: 触れた音の glyph、星座、文字フラグメント
- **装置の窓（画像コンテナ）**: 中央〜下に 200×100 の小窓。葉脈ドット絵が呼吸／触れた瞬間に波紋が走る
- **更新間隔**: 〜150ms（6〜7fps）→ 手書きアニメ的な趣
- **残響表現**（§6.2 と連動）: 過去の振る舞いの glyph がフェード残留し、新しい glyph と重なる

#### archetype ごとの視覚 hint

| | 背景テキスト | 前景 glyph | 装置の窓 |
|---|---|---|---|
| **自然**（★最初） | 霧・点描の漂い | 葉っぱ・苔的グリフ | 葉脈ドット絵の呼吸 |
| **都市** | 電波ノイズ風・グリッド | 直線・回路図的 | 信号波形・走査線 |
| **水辺** | 滲み・水位線 | 同心円・波 | 水紋ドット絵 |

### 7.4 G2 とブラウザの棲み分け

| | 役割 | 美学 |
|---|---|---|
| **G2（眼前・私的・緑モノクロ・576×288）** | **手紙**（手元の私的なメッセージ） | テキスト／グリフ寄り・ASCII signals |
| **ブラウザ（PC全画面・自由）** | **景色**（世界の大きな俯瞰） | sonar／呼吸／星座などゆったり |

> 動画では**この2つの異なる窓を交互に見せる編集**ができ、画面構成が豊かになる。同じ情報の劣化版ではなく、**質の違う 2 つの窓**を持つことに意味を持たせる。

## 8. やらないこと（スコープ外）

- 展示前提の演出 / プロダクトとしての継続利用設計
- DJ/VJ 的スペクタクル
- 派手な反応型ビジュアル
- 完全自動 GPS（必ず手動上書き経路を残す）
- 一般リリース／配布
- 「都市の音楽」を別ジャンル（hip-hop 等）に切り替えるような違和感の大きい palette 切替
- **一回のアクションの演出が完結してから次が始まる**「ぶつ切り」の構成 → §6.2 の残響モデルで連続させる

## 9. 制作ワークフロー（叩き台）

### Step 0. G2 SDK 着手前の確認

> 最新版を確認してから G2 開発に入る（メモリのルール）。

- npm 最新（2026-05-26 時点）: SDK **0.0.10** / CLI **0.1.13** / Simulator **0.7.3**
- 手元: SDK 0.0.9 / CLI 0.1.11 / Sim 0.7.2
- 0.0.9 → 0.0.10 は同年4月のパッチ。changelog 確認し、API 変更が無ければ手元のまま継続も可
- Simulator は **`--automation-port`** が追加されており、撮影編集に使える可能性あり（HTTP 経由で screenshot 取得等）

### Step 1. 事前ロケハン＋通電テスト

- ノートPC + Playtron + `npm run bridge` を持って候補地（公園）へ
- クリップを当て、触ってコンソールに `noteon` が出るか確認
- 「ここはダメ／ここは強い」を分けて本番に反映
- 手の乾燥対策（保湿クリーム / 軽く湿らす）を撮影チェックリストに含める

### Step 2. 撮影前セットアップ

- （GPS フェーズ移行時のみ）GPS 許可、archetype 自動判定 → 必要に応じて手動上書き
- G2 接続、Playtron クリップ設置

### Step 3. 撮影本番

- 1 地点 15〜20 秒。世界の呼吸（潮）と触れの掛け算を捉える
- G2 のグリフが切り替わる瞬間や、ブラウザの俯瞰も別撮りすると編集が豊かに

### Step 4. 編集

- 3 地点を繋ぐ。場面切替の瞬間に「**チューニングが変わる感じ**」（信号のザザッ）を入れると、世界の別チャンネルに合わせ直す儀式感が出る
- **GPS の判定モーメント**（`scanning… → detected:`）を冒頭に入れるとドキュメンタリー性が増す（archetype 自動判定実装後）

### 撮影方法（G2 のキャプチャ）

G2 にカメラはない。**スマホで外撮り ＋ シミュレーターを画面録画して編集で交互**にする方針:

- 現地に Mac/PC を持参 → `npm run bridge && npm run dev` + `evenhub-simulator http://localhost:5173`
- シミュレーターを **macOS 画面録画**（⌘+Shift+5）→ HD で G2 ビュー
- 同時にスマホで装着者と環境を撮る
- 編集で交互に切る ＝「装着者の姿」と「装着者が見ているもの」が同居
- **詳細な準備は撮影テスト時に詰める**

## 10. 未解決の論点（open）

- **撮影候補地の最終確定**: 事前ロケハン＋通電テストで決める
- **「世界の時計」の周期パラメタ**: 何分のスパンを基準にするか、何種類の周期を重ねるか
- **インタラクション分析の閾値**: 「短い／連続／長押し」の境界値（300ms / 500ms / 3s は叩き台）
